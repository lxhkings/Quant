# Web UI 体验重构设计

日期：2026-05-31
范围：`quant/web/` UI 层重构（流程重排 + 引导）。计算逻辑（`viewmodel.py`、`select/`、`report/`、`validate/`）零改动。

## 1. 背景与问题

现有 Web UI 是单文件 `app.py`（约 100 行 `if/elif`），用 sidebar radio 切 6 个页面：因子工坊、多因子合成、holdout 闸门、历史、选股器、批量排行榜。

用户反馈核心痛点：不知道各功能干啥、流程不清楚、设计不合理。逐条定位：

| # | 问题 | 证据 |
|---|---|---|
| 1 | 6 页平铺无顺序、无流程感 | `PAGES` 顺序乱：历史第4、选股器第5、排行榜垫底 |
| 2 | 无首页/无引导，冷启动直接落工坊 | 无 landing |
| 3 | 黑话无解释（lookback/skip/IC/ICIR/DSR/共线性/分位等裸露） | `app.py` number_input 无 help |
| 4 | 无关参数恒显（momentum 只用 lookback/skip，却照显 window/horizon） | `_factor_params` 与 UI 不一致 |
| 5 | 输出是裸 JSON dump（合成页 `st.json`），无通俗结论 | `app.py` 合成分支 |
| 7 | 破坏性操作（holdout 一次性锁定）埋在普通第3 tab | `app.py` holdout 分支 |
| 8 | 数据模式 research/full 全局裸选，无解释 | `app.py` sidebar |

> 病根一句话：这是 6 个孤立工具的堆叠，不是一条研究流水线。功能其实齐全且正好对应专业 alpha 研究管线，只是没把"顺序、为什么、下一步"讲出来。

（注：原编号 #6「跨页无状态」不在本次范围，见 §6 非目标。）

## 2. 目标与定位

- **用户定位**：自用研究。目标是高效跑因子研究全流程（扫→测→合成→定稿→选股），自己产出选股清单。
- **重构深度**：流程重排 + 引导为主。UI 层动，计算逻辑不动。
- **成功标准**：
  1. 冷启动有首页，一眼看懂平台干啥、流程顺序、该先点哪。
  2. sidebar 按专业漏斗顺序排列并编号。
  3. 每个步骤页有「这步干啥 / 上一步 / 下一步」。
  4. 所有黑话参数有 `help` 白话解释。
  5. 合成页消除裸 JSON dump，改用指标卡 + 通俗结论。
  6. 工坊页参数随因子显隐。
  7. 现有 128 个 pytest 用例不受影响（逻辑未动）。

## 3. 专业流程模型

alpha 研究是一个漏斗：想法多 → 层层筛 → 少数活下来 → 组合产出。现有 6 页正好覆盖完整管线，只需排成漏斗：

```
想法多 ┌─────────────────────────────────────────────┐ 信号少
       │ ①扫描  →  ②工坊  →  ③合成  →  ④闸门  → ⑤选股│
       │  粗筛     精测      增强      终验锁定   产出 │
       └─────────────────────────────────────────────┘
                 📒 台账：全程记录，贯穿每一步
```

## 4. 架构与文件结构

采用 streamlit 原生多页机制（`pages/` 目录，文件名数字前缀自动排序成漏斗顺序并生成 sidebar）。

```
quant/web/
├── app.py              # 改为「🏠 总览」首页（漏斗图 + 5 步说明 + 快速入口卡）
├── pages/
│   ├── 1_①_因子扫描.py      # 原"批量排行榜"
│   ├── 2_②_因子工坊.py      # 原"因子工坊"
│   ├── 3_③_多因子合成.py    # 原"多因子合成"
│   ├── 4_④_holdout闸门.py   # 原"holdout闸门"
│   ├── 5_⑤_选股器.py        # 原"选股器"
│   └── 6_📒_台账.py         # 原"历史"
├── _shared.py          # 新：步骤导航条、术语 help 文案、数据模式说明
└── viewmodel.py        # 不动
```

决策：

| 项 | 选择 | 理由 |
|---|---|---|
| 多页机制 | streamlit 原生 `pages/` | 官方方案、自动 sidebar、每页独立文件职责清晰 |
| 数据模式选择器 | 移到每页 sidebar 顶（`_shared` 统一渲染）+ ⓘ | 原生多页下全局 radio 不跨页保留，须每页渲染 |
| 台账 | 独立页，不编号（用 📒） | 它贯穿全程，不属漏斗某一步 |

取舍：原全局 `mode` selectbox 在原生多页下不自动跨页保留。本次范围是「流程重排 + 引导」，不引入 `session_state` 跨页同步（属于状态串联范围，见 §6）。故 mode 每页各自选，默认 research——与现状行为一致，只是位置规整。

## 5. 详细设计

### 5.1 首页（🏠 总览，`app.py`）

纯展示，无计算。回答三问：平台干啥、流程啥、先点哪。

- 顶部：一句平台简介 + 漏斗图（markdown 等宽块或 `st.columns` + 箭头排，不引图表库）。
- 6 张卡（`st.columns`），每张三行：**干啥** + **何时用** + 跳转按钮（`st.page_link`）。
- ④闸门卡带 ⚠ 标记破坏性。

卡片文案：

| 卡 | 干啥 | 何时用 |
|---|---|---|
| ①因子扫描 | 一键体检全库因子，按 ICIR 排序，找出值得深挖的 | 开局粗筛 |
| ②因子工坊 | 单因子深度体检：预测力/单调性/多空三关红绿灯 | 盯一个因子 |
| ③多因子合成 | 几个因子加权合成，查共线性，回测 | 组合增强 |
| ④holdout闸门 | ⚠ 定稿因子最终验证，仅一次，跑后锁定 | 即将上线 |
| ⑤选股器 | 多因子加权打分，产出今日买入池 | 要选股清单 |
| 📒台账 | 看历史每次试验记录 | 回溯 |

### 5.2 通用骨架（`_shared.py`）

提供 3 个函数：

| 函数 | 作用 |
|---|---|
| `step_header(step_no, title, what, prev, next)` | 渲染「第 N 步 / 5 + 上下步 page_link + 这步干啥 `st.info` 框」 |
| `mode_selector()` | sidebar 数据模式选择器 + ⓘ 说明，返回 mode |
| `GLOSSARY` | 术语→白话 dict，给参数 `help=` 用 |

页头部布局：

```
第 2 步 / 5 · 因子工坊
[◀ ① 因子扫描]              [③ 多因子合成 ▶]
┌─ 这步干啥 ──────────────────────────────┐
│ 对单个因子做 IC + 分位 + 多空体检，看它 │
│ 有没有预测力。跑完看顶部红绿灯三关。    │
└─────────────────────────────────────────┘
```

sidebar 数据模式：

```
数据模式 [research ▾] ⓘ
  research: 留出近2年不看，防偷看
  full: 用全部数据(仅最终选股用)
```

术语表（`GLOSSARY`，给 `help=` 用）：

| 术语 | help 白话 |
|---|---|
| lookback | 回看多少天算动量，常用 252（约1年） |
| skip | 跳过最近几天，避开短期反转，常用 21 |
| window | 计算窗口天数，常用 200 |
| horizon | 预测未来多少天的收益，常用 21 |
| IC | 因子值与未来收益的相关性，越高预测力越强 |
| ICIR | IC 均值 / IC 波动，>0.5 算不错 |
| 分位档数 | 按因子值把股票分几组看单调性，常用 5 |
| 共线性 | 因子间太像（相关>0.7）会重复下注 |
| DSR | 抗过拟合指标，扣掉多次试验的运气成分 |
| 行业中性化 | 去掉行业影响，只看因子本身 |

### 5.3 逐页改动

每页 = `step_header` + `mode_selector` + 原操作。`viewmodel` 调用全不变。

**① 因子扫描**（原批量排行榜）：header 第1步，无上一步，下一步→工坊。dataframe 保留；补结论引导「ICIR 高的几个，去 ②工坊 深挖」。

**② 因子工坊**：header 第2步，①↔③。**修参数显隐**：选 momentum→只显 lookback/skip；其余因子→只显 window（修问题 #4）。`horizon/quantiles/neutralize` 通用保留，全加 `help`。输出报告卡 markdown 保留。

**③ 多因子合成**：header 第3步，②↔④。**杀 JSON dump**（修问题 #5）：`st.json(weights)`→权重条形图/表；`st.json(metrics)`→4 个 `st.metric` 卡（年化/Sharpe/回撤/月胜率）；顶部加通俗结论（共线性有无预警 + Sharpe 达不达标）。nav 图保留。

**④ holdout 闸门**：header 第4步，③↔⑤。破坏性强化：顶部红色风险条 + 现有二次 checkbox 确认保留。结果 markdown 保留。错误：已消耗 `RuntimeError`→`st.error`（现状保留）。

**⑤ 选股器**：header 第5步，④↔（无下一步，指回 📒台账）。bar chart + dataframe 保留；顶部补「今日买入池 N 只，截面日 {as_of}」，归一权重从 caption 提为小表。

**📒 台账**：header 简化（仅标题 + 「全程试验记录」一句，无编号）。dataframe 保留。

## 6. 非目标（YAGNI）

- 跨页状态串联（工坊选的因子自动带到合成/选股）。属更深范围，本次不做。
- 单页向导（wizard）。流程感最强但超范围，且自用老手不需强牵引。
- UI 层自动化测试。streamlit UI 本无测试，自用工具不新增。

## 7. 测试与验收

- `viewmodel`/`select` 逻辑未动，现有 128 个 pytest 用例不受影响（`uv run pytest -q` 仍全绿）。
- 手动验收清单：
  1. 6 页能开，sidebar 按 ①②③④⑤ + 📒 顺序排列。
  2. 首页漏斗图 + 6 卡显示，`page_link` 能跳转。
  3. 每步骤页头部显示「第 N 步 / 5 + 上下步链接 + 这步干啥」。
  4. ②工坊：选 momentum 显 lookback/skip，选其他因子显 window。
  5. ③合成：无裸 JSON，显 metric 卡 + 通俗结论。
  6. 参数 hover 显示 help 白话。
